｛各ファイルの概要｝
.index:初めのスタート画面です。ゲームスタートボタンを実装しています
.Select Difficulty.html：難易度選択画面です。ｈ２にはｊｓから送られてきた問題が記載されます。resultには条件分岐によってでた（正解○）（不正解×）が記載されます
.script.js:問題が動くロジック。内容は概要の下に記載
.css:画面のデザイン

｛コードが動くロジック｝
(１)クイズのデータ（quizData）初級・中級・上級ごとの「問題・選択肢・正解」のデータを、1つの大きな箱（オブジェクトの配列）にまとめて保存しています。
（２・３）ゲームが始まるときの動き難易度ボタンが押されると、ゲームを始める命令（startQuiz()）が動きます。難易度選択ボタンを画面から消して、代わりにクイズ画面を出現させます。クイズを「1問目（インデックス番号0）」にセットし、選ばれた難易度のデータを読み込んで、問題を画面に出す命令（displayQuestion()）を呼びます。
（４）クイズが終わったときの動きすべての問題を解き終わると、終わりのテキストと合計スコア（点数）の結果を表示します。3つの選択肢ボタンを消して、代わりに「難易度選択画面に戻る」ボタンを出現させます。このボタン（gotoHome）を押すと、解いた問題の数やスコアを「0」にリセットして、最初の画面に戻れます。
（５）問題を画面に出す＆ボタン連打の防止問題文（questionElement.textContent）を画面に書き込み、3つのボタン（buttons.forEach）に選択肢の文字を順番に書き換えて表示します。ボタンを1回押したら、他のボタンを一時的に押せなくする（disabled=true）という「条件分岐（じょうけんぶんき）」の仕組みにしています。これで、ボタンを連打されても問題が勝手に次へ進まないようにガードしています。プレイヤーが選んだボタンが正解なら、スコア（点数）が1点増えます。
（６）1秒待って次の問題へ答えを押したあと、タイマー（setTimeout）を使って1秒間だけストップさせます。その間に画面に〇や✕の結果を出します。1秒経ったら〇✕のテキストを消し、カウントを1つ増やして（++）つぎの問題を表示します。
